What is a Git ?


Git is a very popular among developers Git is a distributed version control system for tracking file changes git helps us to work with multiple people, on the same file on the same time, which is a superpower when it comes to developing something, It allows you to create your own working copy of project on your local machine and allows you to make it remote so other team member can read and update it.

In this blog, will discuss the basic Git workflow and basic CLI commands of git for a beginner to start with. Let's jump right in.



Basic Idea of Git Workfolw


Git Is a version control It is a software use to track the local file. Gitlab and Github are Remote code repository where all other developer push there git initiallized local repo And with the help of github and gitlab developers can work collaboratively.

Git basically work on 3 stages first step is working stage where we can locally modified our codes, after that we use add command so our code goes from working area to stagging area From stagging area we do final commit, which is the last step and it puts a snapshot of whatever was there in the staging area as a permanent commit/version.
Hence the name Version Control, Then obviously we can do push command to push all our commits into the remote repo present in GitHub or GitLab. .

Basic Git Commands

Following are some basic CLI git commands to get started as beginner

1) Git init -

This command is used to initialized an empty git repository in the current directory.

$ git init

2) Git Status -

This command gives the status of files changed and those which still need to add or commit as untracked, modified, or staged.

$ git status

3) Git Add -

Using this command you can add your code changes to staging area, you can add more than one file at same time

$ git add

#to add a one file

$ git add .

#to add all untracked files together into the staging area

4) Git Commit -

This command is used to commit changes with comments locally not to the remote repository. This will take files from staging area and saves the snapshot to the project history and completes the change-tracking process.

$ git commit -m "message"

5) Git log -

This command lists all the previous commits with details such as commit id , commit message, and date.

$ git log

6) Git Branch -

This command is used to work with the branches. We can create different branches to work on a feature and merge it later with the main branch.

$ git branch

#gives list all the branches

$ git branch [branch-name]

#To create new branch

7) Git Checkout -

This command is used to switch between different created branches in repository.

$ git checkout [branch-name]

8) Git Clone -

This command is used to to clone remote repository into local machine. This includes all the projectâ€™s files, history, and branches.

$ git clone [remote url]

9) Git Push -

This command is used to Push the changes to the remote repository.

$ git push origin master

#push all changes in the master branch from local to remote repository

$ git push origin [branch-name]

#push all changes in the a specific branch to remote repository

10) Git Pull -

This command is used to fetch content from a remote repository to the local working directory.

$ git pull

11) Git Merge -

This command is used to merge a different branch into your working branch or to master/main branch.

$ git merge [branch-name]

12) Git Diff -

This command is used to Show difference between previous changes and new changes.

$ git diff
